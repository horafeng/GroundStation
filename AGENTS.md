# 无人机地面站 Demo

## 项目目标

本项目是一个运行于 Windows 的无人机地面站 Demo。

软件负责：

1. 通过 UDP 接收雷达数据。
2. 解析雷达自身状态与目标航迹。
3. 在界面中显示多个雷达目标。
4. 由操作人员人工选择目标无人机。
5. 根据任务模式，通过 UDP 循环发送目标信息。
6. 目标丢失后，继续发送最后一次有效坐标。

无人机收到信息后的飞行控制逻辑不属于本项目。

## 目录边界

- D:\Drone 是只读资料目录。
- 禁止修改、删除、移动或重命名 D:\Drone 中的任何文件。
- D:\GroundStation 是唯一允许写入代码、文档、测试和配置的目录。
- 禁止在 D:\GroundStation 之外生成项目文件。

## 技术要求

- 操作系统：Windows 10 / Windows 11
- Python：3.11
- UI：PyQt5
- 雷达通信：UDP
- 无人机通信：UDP
- 默认循环发送频率：5 Hz，即每 200 ms 一帧
- 发送频率必须可配置
- 所有源代码使用 UTF-8
- 使用类型注解
- 网络接收不得阻塞 UI 线程
- IP、端口和频率不得写死在业务代码中

## 任务模式

暂定编码：

- 0：待命，在当前位置悬停
- 1：起飞，垂直起飞至 5 米并悬停
- 2：跟踪，持续接收最新目标经纬高
- 3：返航，返回 Home 点上方 5 米并悬停
- 4：降落，返回 Home 点并降落

地面站只发送模式编号，具体执行由无人机侧程序负责。

## 雷达数据要求

重点解析目标航迹数据，包括：

### 雷达自身信息

- GPS 经度
- GPS 纬度
- GPS 海拔高度
- 卫星个数

### 每个目标的信息

- 航迹显示编号
- 航迹绝对编号
- GPS 经度
- GPS 纬度
- 目标高度
- 目标类型
- 速度
- 方位
- 航迹消除或失效状态

程序内部优先使用航迹绝对编号持续关联目标。

## 人工选择目标

- 地图或航迹列表显示所有有效目标。
- 不自动认定哪个目标是无人机。
- 操作人员人工选择目标。
- 任务执行过程中选择另一个目标时，必须弹出确认框。
- 确认内容示例：
  是否将跟踪目标由航迹 7 切换为航迹 12？
- 用户确认后才能切换目标。

## 目标丢失

目标丢失后：

- 保留最后一次有效经纬度和高度。
- 继续以当前频率发送最后坐标。
- 不自动停止。
- 不自动将附近的新航迹识别为原目标。
- 报文包含目标坐标时间戳。
- 报文包含坐标是否为实时值。
- 报文包含目标丢失持续时间。

目标高度目前暂定为相对地面高度，单位为米。

这一点属于待实物验证的项目假设，必须在代码和文档中明确标注。

## 已确认的核心实现规则

- 航迹仓库必须以航迹绝对编号为主键；显示编号只用于显示，允许复用和变化。
- 不自动过滤类型3，不自动选择最近目标，不做邻近航迹自动重关联。
- 目标消除或超过超时阈值后保留最后一次有效坐标。
- 丢失状态和发送周期使用单调时钟；对外Unix时间戳与单调计时分离。
- 超时阈值只决定何时把坐标标为非实时；丢失持续时间必须从最后一次有效坐标的
  单调接收时刻起算，不能从超时阈值时刻或收到消除标志的时刻重新从0累计。
- 航迹未更新超过2000ms判为丢失是可配置的Demo临时假设，尚未经过真实雷达更新频率验证。
- 程序启动时不恢复选择；初始目标无效，零经纬度不得被当作有效目标。
- 模式改变、首次选择和确认切换目标产生立即发送请求。
- 周期帧与立即帧必须在同一发送worker内串行编码和发送，共享uint32序号。
- 当前64字节、小端、CRC32/IEEE无人机报文只能称为Demo临时协议，不能称为正式无人机协议。
- `DroneUdpTransport`只接收`bytes`，不得了解任务或航迹字段。
- 当前阶段不开发完整PyQt5界面、雷达盘、MAVLink、飞控接入或任何无人机飞行动作。
