# 无人机地面站 Demo 需求基线 v0.1

- 状态：待确认
- 日期：2026-07-15
- 适用系统：Windows 10/11
- 技术栈：Python 3.11、PyQt5、UDP

## 1. 项目边界

地面站只负责：接收雷达 UDP、按正式协议解析、显示多个目标、由操作员人工选择目标、生成任务状态、通过可替换编码器循环发送 UDP。

明确不负责：无人机解锁、飞控控制算法、飞行路径执行、自动目标识别、自动目标重关联、光电设备控制、在线地图接入和 MAVLink 飞控实现。

数据链路：

`雷达 -> UDP -> Windows 地面站 -> UDP -> 无人机端接收程序`

## 2. 强制目录规则

- `D:\Drone` 只读，禁止修改、删除、移动、重命名。
- 所有代码、配置、文档、测试和运行说明只写入 `D:\GroundStation`。
- 雷达解析不得以旧源码或经验替代正式协议。

## 3. 雷达接收与解析

### R-RADAR-001

接收 UDP 的阻塞操作不得运行在 UI 主线程。

### R-RADAR-002

解析器必须依据正式协议识别“目标航迹数据”，校验帧头、长度、目标数、帧尾和校验和；异常帧必须被拒绝并记录原因。

### R-RADAR-003

雷达自身状态至少输出：GPS 经度、GPS 纬度、GPS 海拔高度、卫星数、接收时间和数据有效性。

### R-RADAR-004

每个航迹至少输出：显示编号、绝对编号、经纬度、目标高度、目标类型、速度、方位、距离、协议消除状态、雷达时间字段和本机接收时间。

### R-RADAR-005

网络字节序在确认前不得写死为协议事实；解析器应由配置或已确认协议版本决定，而不是对生产报文“试两种取能解析者”。

## 4. 航迹管理与人工选择

### R-TRACK-001

UI 显示所有协议有效的目标，不得只显示类型 3；目标类型只作为操作员判断信息。

### R-TRACK-002

UI 显示航迹显示编号；内部持续关联优先使用航迹绝对编号。显示编号只用于面向操作员的标识。

### R-TRACK-003

不得自动选择“无人机”、最近目标或附近新航迹。

### R-TRACK-004

任务进行中选择另一航迹时，必须弹窗确认：`是否将跟踪目标由航迹7切换为航迹12？`。取消后保持原目标。

### R-TRACK-005

确认选择或切换目标后立即生成并发送一帧。

## 5. 任务模式

| 编码 | 名称 | 地面站语义 |
|---:|---|---|
| 0 | 待命 | 只发送模式和当前目标快照；无人机侧负责悬停 |
| 1 | 起飞 | 只发送模式；无人机侧负责起飞至相对地面 5 m |
| 2 | 跟踪 | 持续发送所选目标最新或最后有效坐标 |
| 3 | 返航 | 只发送模式；无人机侧负责返回 Home 上方 5 m |
| 4 | 降落 | 只发送模式；无人机侧负责返回 Home 并降落 |

模式切换后必须立即发送一帧。五种模式下都保留最后一次有效目标坐标；从未选择过目标时必须明确编码为“目标无效”。

## 6. 目标丢失

### R-LOST-001

所选目标正常且未消除时，更新最后有效经纬度、高度和坐标时间戳，`is_realtime=1`、`lost_duration=0`。

### R-LOST-002

所选目标被协议标记消除，或超过可配置的雷达更新判定窗口未出现时，不清空坐标；继续发送最后有效坐标，`is_realtime=0`，丢失持续时间单调增加。

### R-LOST-003

丢失后不得自动关联附近航迹；只有操作员重新选择并确认才可改变目标绝对编号。

### R-LOST-004

目标高度暂按“相对地面高度，单位米”处理，但这是未实物核对的业务假设，UI、代码注释、配置和文档必须显式标记。

## 7. 无人机 UDP 发送

- 默认 5 Hz（200 ms），频率必须可配置。
- 模式切换和目标确认后立即发送，不等待下一个周期。
- 报文序号单调递增并在 UI 显示；溢出行为由临时协议文档明确。
- UDP 发送与协议编码分离；UI、雷达解析、航迹管理不得导入具体编码器实现。
- 正式无人机协议未确定前，只能提供单独的临时编码器，并把字节数、字节序、校验等全部标为 Demo 约定。

建议的编码器输入是与二进制无关的 `MissionSnapshot`，包含：协议版本、序号、无人机 ID、模式、目标有效标志、绝对编号、坐标、高度、坐标时间戳、实时标志、丢失持续时间和备用字段。

## 8. UI 最小范围

- 雷达连接状态、雷达自身状态。
- 离线二维雷达图或简易地图。
- 多目标航迹列表及目标类型、速度、方位、距离、更新时间、消除状态。
- 当前目标详情、绝对编号和显示编号。
- 五个任务模式按钮及当前模式。
- 雷达本地 IP/端口、雷达来源过滤设置。
- 无人机目标 IP/端口、发送频率设置。
- 当前报文序号、目标有效/实时/丢失状态和丢失持续时间。
- 通信日志和完整/截断可配置的十六进制日志。

## 9. 质量与生命周期

- 使用数据类、枚举和类型注解。
- UDP 接收、雷达解析、航迹管理、选择、模式、编码、发送、UI、配置和日志解耦。
- 程序退出时停止计时器/线程，关闭 Socket，并对线程执行有界等待。
- 提供独立雷达模拟发送器、独立无人机模拟接收器。
- 提供协议单元测试、航迹生命周期测试、编码器契约测试、UDP 集成测试和安全退出测试。
