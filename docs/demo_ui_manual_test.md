# PyQt5 Demo手动演示说明

## 1. 演示边界

本流程只演示雷达UDP、严格解析、航迹生命周期、人工选择、任务模式和64字节
Demo临时协议发送、交互地图和视频占位视角。不包含MAVLink、真实飞控、无人机飞行动作、
自动目标识别、自动重关联、真实视频解码或完整离线地图。

地图投影和底图仅用于Demo显示。原始雷达经纬度原样进入航迹仓库和MissionSnapshot，
地图拖动、缩放和显示投影不会修改发给无人机的坐标。

## 2. 环境准备

在PowerShell中执行：

```powershell
cd D:\GroundStation
py -3.11 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
```

默认端口：雷达监听`127.0.0.1:6000`，无人机接收`127.0.0.1:7000`。

## 3. 三个终端

终端A，启动无人机模拟接收器：

```powershell
cd D:\GroundStation
.\.venv\Scripts\python.exe tools\drone_receiver_simulator.py --host 127.0.0.1 --port 7000
```

终端B，启动地面站（唯一推荐入口）：

```powershell
cd D:\GroundStation
.\.venv\Scripts\python.exe app.py
```

终端C暂不执行命令。

## 4. 雷达接收与多目标显示

1. 在主界面确认监听与发送摘要；点击“系统设置”可在独立窗口中查看和修改完整
   设置。窗口一级页签为“系统设置”“人机校验”，系统设置内含雷达、无人机、协议、地图
   和视频页签。监听地址、来源过滤和
   字节序需要停止并重新启动雷达监听才生效；无人机端点、ID和频率需要停止并重新启动
   发送才生效；航迹超时阈值点击“应用”后立即生效。
2. 点击“运行检查”，在“通信”页点击“启动雷达监听”。主界面监听状态应变为“监听中”，
   数据状态初始为“尚未收到数据”。
3. 在终端C启动四个不同类型的持续移动目标：

```powershell
cd D:\GroundStation
.\.venv\Scripts\python.exe tools\radar_simulator.py --scenario multi-moving --host 127.0.0.1 --port 6000 --count 0 --interval-ms 200
```

4. “运行检查”的通信页应显示雷达经纬度、海拔和卫星数，顶部“航迹”入口显示4；
   地图以雷达/Home为中心。
5. 地图应显示全部目标、尾迹和显示编号7/12/21/33；类型3不会被自动选中或特殊高亮。
6. 观察“最近雷达报文”“雷达自身更新时间”和目标坐标时间戳，均应显示本机当前年份，
   不应显示1970年。
7. 主界面不得出现固定航迹、日志或通信诊断区域，地图/视频应占满状态条以下空间。
8. “运行检查”窗口的航迹、通信、运行日志、报文和协议错误页签均应为深色底且文字清晰；
   航迹表完整填充页签，滚动条贴合表格边界。

## 5. 人工选择、发送和模式

1. 点击地图航迹7或在“运行检查”的航迹页点击航迹7。首次选择直接成功，地图、表格和
   右上目标卡片同步；点击卡片可稳定展开和收起明细。
2. 在“运行检查”的通信页点击“启动无人机发送”。终端A应接近5Hz收到64字节报文，
   `target_valid=1`。
3. 点击航迹12，确认框文字应为“是否将跟踪目标由航迹7切换为航迹12？”，
   并显示双方绝对编号和坐标。
4. 点击取消，终端A继续显示航迹7的绝对编号`100007`。
5. 再点航迹12并确认，应立即出现一帧`reason=target_switch_confirmed`对应的新目标，
   随后周期帧继续发送绝对编号`100012`。
6. 依次点击待命、起飞、跟踪、返航、降落；终端A应依次解码模式0、1、2、3、4。
   每次实际模式改变都会请求立即帧。
7. 运行检查通信页的“本机发送成功”只表示本机`sendto`未报错；“接收确认”必须始终是
   “未知（Demo协议无ACK）”，不代表终端A或真实无人机已收到。

## 6. 消除与超时

若当前选择航迹7，可先停止终端C的移动场景，然后发送带消除标志的同一绝对编号：

```powershell
.\.venv\Scripts\python.exe tools\radar_simulator.py --scenario multi-moving-clear --host 127.0.0.1 --port 6000 --count 5 --interval-ms 200
```

UI应保留航迹7及最后有效坐标，状态显示“最后有效坐标（目标丢失）”；终端A中
`target_valid=1`、`coordinate_realtime=0`，丢失持续时间从最后有效坐标的单调
接收时刻开始增加，而不是从消除报文到达时从0开始。

测试超时可重新启动`multi-moving`，选中任意目标后停止模拟器。超过默认2000ms，
同样应转为丢失并继续5Hz发送最后坐标。新出现的其他绝对编号不得替换当前选择。
例如停止更新2300ms且阈值为2000ms时，接收器应显示约2300ms，而不是约300ms。
此时监听状态仍应为“监听中”，数据状态应为“数据超时”，且错误帧计数不应增加。

## 7. 列表滚动与发送接收器关闭

1. 使用多于可视行的航迹数据，选择一条靠前航迹后手动滚动到表格底部。
2. 等待多次200ms界面刷新：滚动位置不得跳回选中航迹，地图右上目标卡片仍持续显示。
3. 确认切换目标后允许定位到新行；周期刷新本身不得改变滚动位置。
4. 关闭终端A的无人机模拟接收器，地面站仍会按频率递增序号，且“本机发送成功”可能
   继续增加；这不是无人机在线或已接收的证据。

## 8. 地图降级与主副视角

1. 在系统设置的地图页选择“在线 OpenStreetMap”并应用；网络可用时显示标准瓦片和
   `© OpenStreetMap contributors`，只加载当前视口。
2. 断网或瓦片服务不可用时，状态提示降级，本地网格/道路/水域示意底图、Home、目标、
   轨迹、点击选择、拖动、缩放和回 Home 仍可用。
3. 单击左下视频画中画，视频成为主区、地图成为画中画；再次点击地图小窗恢复地图主视角。
   左上纵向任务按钮和右上目标卡片的位置、选择及展开状态不得变化。
4. 视频设置为测试图案时只显示测试图案；配置 RTSP 时明确提示本轮未启用解码器。
5. “人机校验”页全部操作不可用，并明确说明未接入飞控双向协议。

## 9. 关闭检查

1. 点击“停止无人机发送”，确认终端A不再收到新帧。
2. 点击“停止雷达监听”。
3. 关闭地面站窗口。
4. 地面站按“发送调度器 → 雷达接收Socket/线程 → 日志资源”顺序退出。
5. 再次启动地面站并绑定6000端口应成功，不应残留线程或Socket。

## 10. 当前临时假设

- 雷达32位字暂按小端序解析、一UDP数据报暂按一完整帧处理，均未经过真实抓包验证。
- 航迹未更新超过2000ms判为丢失，尚未经过真实雷达更新频率验证。
- 目标高度暂按相对地面高度、单位米解释，待实物核对。
- 无人机报文为64字节Demo临时协议，不是正式无人机协议。
