## 2. P0：真实雷达接入前必须确认

### U-001 网络字节序

正式雷达协议未写明 UDP 线上 32 位字的字节序。旧程序同时尝试小端和大端不是协议证据。本阶段按 A-006 实现可配置的小端解析，但真实接入前仍需要厂家书面确认或至少一份已知字段的真实 UDP 十六进制报文。

### U-002 数据报与帧边界

本阶段按 A-007 将一个 UDP 数据报视为恰好一帧。真实接入前仍需确认是否可能一报多帧或出现应用层分片；若存在，必须取得正式定义。

### U-003 雷达坐标基准

协议称“GPS 经度/纬度”，未明确写 WGS84。离线雷达盘可暂不依赖地图坐标系，但无人机报文和未来地图必须确认 WGS84/GCJ-02/其他基准。

### U-004 雷达海拔子字段符号

整字类型为 `Uint`，高 16 位写“海拔高度”，未明确有无符号。旧源码符号扩展不能当作正式依据。

### U-005 目标高度基准

协议只写“目标高度信息，单位 m（short 类型）”，未说明相对地面、相对雷达、海拔或其他。当前相对地面解释只是用户指定的待核对假设。

## 3. P1：航迹生命周期和丢失逻辑前必须确认

### U-006 清除标志发送行为

需确认目标块状态字 Bit31 的清除状态会持续多帧、只发一次，还是与结束帧配合使用。

### U-007 正常更新率和丢失阈值

需获得航迹帧标称频率、最大抖动和雷达忙时延迟，才能确定默认 `track_stale_timeout_ms`。

### U-008 航迹绝对编号生命周期

需确认绝对编号在雷达重启、断链、跨日后是否复用；若复用，内部键应加入雷达会话 ID。

### U-009 时间字段

需确认 DSP 时间、GPS 时刻、目标 GPS 时间、24 位 DSP 时间标志的单位、日滚转和对应关系。坐标时间戳优先使用哪个字段尚未确定。

### U-010 方位角语义

协议称目标角度为 NED 坐标系信息。旧源码又加雷达航向角，缺乏协议依据。需确认方位角是否已经是以真北为零的绝对 NED 方位。

## 4. P1：无人机临时报文前必须由项目方选择

以下是可替换临时协议必须明确、但本轮不得擅定的内容：

- 帧头和帧尾常量。
- 协议版本字段宽度。
- 长度单位（字节或字）。
- 所有整数和浮点字段宽度。
- 整体字节序。
- 经纬度表示（整数缩放或 IEEE 754）。
- 高度类型和范围。
- 时间戳单位/时区。
- 丢失持续时间单位和最大值。
- 序号位宽与回绕规则。
- 校验方式（累加和、CRC16、CRC32 等）。
- 无人机 ID 位宽。

建议在实现前单独提交 `temporary_drone_protocol_v0.1.md` 供确认，并生成一组十六进制示例。

## 5. 产品交互待确认

- “任务执行过程中”从何时开始：任何已选目标，还是仅跟踪模式？建议只要已有选中目标，切换都确认。
- 是否允许显式“取消选择目标”；取消后报文是目标无效还是继续保留最后坐标。
- 程序重启是否恢复上次选中目标。安全起见建议不恢复，启动时目标无效。
- 五模式按钮是否需要二次确认。建议返航/降落至少预留确认策略，但当前需求只强制目标切换确认。
- 发送频率允许范围。建议配置校验范围待联调后确定。
- 雷达来源端口是否固定；旧代码严格匹配来源端口可能在 NAT/设备实现下不成立。

## 6. 历史假设中明确不再沿用的内容

`D:\Drone\目前假设.txt` 中以下假设属于另一项目方向，与当前地面站 Demo 边界冲突：

- UDP 直接承载 MAVLink。
- 地面站直接控制 X6 飞控。
- 自动切换 GUIDED、发送 `SET_POSITION_TARGET_GLOBAL_INT`。
- 地面站负责解锁、起飞、返航或降落的飞控实现。
- 默认将目标高度映射到 MAVLink 高度坐标框架。

当前项目只发送模式编号和目标信息；无人机端如何执行完全不在范围。

## 7. 建议向厂家/项目方索取的最小资料包

1. 雷达航迹 UDP 的 10-30 秒 PCAP，至少包含多目标、新目标、更新和清除。
2. 与 PCAP 同时刻的雷达界面截图或导出，便于已知值对照。
3. 字节序、帧/数据报边界和校验规则的书面确认。
4. 目标高度/雷达海拔基准说明。
5. 航迹更新率、清除标志行为、绝对编号生命周期说明。
6. 无人机端正式接收协议或临时协议评审反馈。

## 8. 风险结论

U-001 和 U-002 已按项目方批准的 Demo 临时假设实现为配置项，但仍未完成真实雷达验证，不能宣称已经能正确解析真实雷达。U-005（目标高度基准）未确认前，任何发往真实无人机的高度都只能用于受控仿真，不能作为实飞依据。
